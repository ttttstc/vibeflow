Definition

什么是 VibeFlow

VibeFlow 是一个< span class="highlight">标准化 AI 工程交付编排框架。
给 AI 铺上轨道，让交付多一点章法、记忆与收尾。

05 / 18

完整交付链路

Think → Plan → Requirements → Design → Build → Review → Test → Ship → Reflect

每一步都有状态文件、规格文档和验证产物，让 AI 可以继续、恢复、审查和交接

10 / 18

Think 阶段

在开始编码之前，先理解问题的本质

  • 澄清需求：用户真正想要解决的是什么？
  • 评估影响：这个改动会影响到哪些部分？
  • 识别风险：有哪些技术债务或潜在问题？
  • 确定范围：这次交付的边界在哪里？
11 / 18

Plan 阶段

制定清晰的执行计划，而不是边做边想

  • Requirements → 需求文档，明确"做什么"
  • Design → 设计文档，明确"怎么做"
  • Plan Review → 评审检查，确保计划可行
  • Spec-Driven → 先写规格，再进入实现
12 / 18

Build 阶段

自动续跑：进入 build-init 后，系统默认自动推进

Build Init → TDD → Quality Gates → Feature Acceptance → Spec Review

每个检查点都是一次质量门，确保代码符合预期

13 / 18

Ship & Reflect

Ship

  • 发布验证：确保可部署
  • Release Notes：变更记录
  • 增量请求：下一个功能的起点

Reflect

  • 复盘总结：这次有什么收获？
  • 改进建议：下次如何做得更好？
  • 知识沉淀：经验变成文档
